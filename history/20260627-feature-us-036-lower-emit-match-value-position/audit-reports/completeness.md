# Completeness Audit — US-036

## Findings

### MINOR — Statement-position arm with a block body
The spec phrases value-position arm bodies as expressions, but statement-position
arms can be calls or blocks (`Status.Pending => startOnboarding()`). The existing
`armBody` helper already handles block/stmt/expr bodies, so no spec change is
required; noted for the implementer.

### MINOR — Guard variable naming
The spec says references resolve to "the type-switch guard variable" without
naming it. The backend uses a scope-aware gensym (`v`, US-035), not a fixed
`__goal_v`. This is intentional (exact golden parity is US-042) and does not
affect behavioral conformance. Not blocking.

### MINOR — New corpus case content
The spec requires a new value-position-match case but does not pin its exact
source. Any enum value-position `match` (e.g. a `return match`/`var = match` that
exercises a payload field) satisfies AC; left to the implementer.

## No CRITICAL or MAJOR findings.

The four 02-match cases concretely pin all happy-path behaviors (statement, rest
arm, return, var). Error handling (unsupported shapes still error) is specified.

## Assumptions
- The new value-position-match corpus case is a transpile case with a
  `.go.expected` golden; it is added under `testdata/` and the manifest is
  regenerated by `cmd/corpus-gen`.
- "Behavioral tier" means temp-module `go build` + `go vet` via
  `corpus.RunCompile`, consistent with US-032..035.
