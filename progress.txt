# Progress Log — goal AST front-end rewrite

## Codebase Patterns
- Zero-dependency project: tests use stdlib `testing` only (no testify).
- Verify with `task check` (go vet + full `go test ./...`) and `task build`.
  A PostToolUse hook runs `task check` after every Edit, so a multi-file refactor
  reports transient "undefined: X" / "imported and not used" failures until the
  LAST edit lands — expected; only the final green state matters.
- sema is the one live checker. sema.Diagnostic.Pos is a token.Pos carrying
  Line/Col, so there is no OffsetToPosition step for diagnostics — read
  d.Pos.Line / d.Pos.Col directly. For a raw byte offset (not a diagnostic) use
  token.OffsetToPosition(src, off). Single-file check: sema.Analyze(src); package:
  sema.AnalyzePackageInDir(srcs, dir) / ...InDirWith(srcs, dir, resolve).
- Diagnostic severity unifies on sema.Severity (sema.Error / sema.Warning).
  A package that already imports `go/token` as `token` cannot also alias
  goal/internal/token — keep a tiny local offset->line/col helper instead.
- When the diagnostic-code set changes, two goldens must follow: the guide catalog
  (internal/guide/catalog.go, asserted against scanned source by guide_test
  TestDiagnosticCatalogMatchesSource) and AI-KNOWLEDGE-BOOTSTRAP.md (regenerate
  with `go run ./cmd/goal ai > AI-KNOWLEDGE-BOOTSTRAP.md`).
- internal/textedit is the home for lexer-independent text/type utilities
  (Replacement, Splice, BaseType, IsIdent, IsLineStart, NextNewline, LeadIdent,
  SplitAssign, IsStmtKeyword, ZeroLit). It imports only sort/strings/unicode —
  never text/scanner. The lexer-based helpers (Lex, Token, Match*, ScanFuncs,
  CalleeKey, FirstBodyBrace, ParamsClose, TopLevelComma, IsBareQuestionStmt)
  stay in internal/scan, which now imports textedit for the pure bits.

## [2026-06-28T18:05:00Z] - US-001
- Proved US-001 "Lower Option construction in value positions": Option.Some/None
  now lowers to its *T pointer encoding wherever an Option value is PRODUCED — a
  var/assign RHS (`x := Option.Some(v)` -> `x := &v`), a call argument, a
  struct-literal field, and slice/map-literal elements — not only at a direct
  return or as a Result.Ok payload. Previously these reached the generic expr path
  and emitted `Option.Some/None` verbatim, leaving `Option` undefined in the Go.
- Mechanism: a single interception at the TOP of emitter.expr (the one
  value-emission seam all value positions route through). optionConstruction
  (lower.go) classifies an Option construction as "none" (-> nil), "some-addr"
  (single Ident arg -> &arg), or "some-box" (any other single arg -> goalSome(arg)).
  A non-addressable Some cannot be address-taken as a pure expression and the
  single-pass emitter cannot hoist a temp there, so it boxes through a generic
  helper `func goalSome[T any](v T) *T { return &v }` (optionPrelude), appended at
  end-of-file (Go decls are order-independent) gated by the runtime
  emitter.usedOptionHelper flag — emitted only when actually used. Package mode
  surfaces the flag out of emitFileWith and emits one shared goal_options.go.
- Files: internal/backend/lower.go (optionConstruction, optionPrelude),
  internal/backend/emit.go (usedOptionHelper field, tryOptionValue, expr()
  interception, file() end-append, emitFileWith returns usedOption),
  internal/backend/package.go (shared goal_options.go), internal/backend/backend_test.go
  (TestASTEngineLowersOptionInValuePositions). Existing
  TestASTEngineLowersNestedOptionInResult + whole corpus/pipeline tier stay green
  (the return/Result.Ok optionValueExpr path is untouched).
- Learnings: see the new Codebase Patterns entries (single expr() seam, runtime-flag
  end-of-file helper vs over-counting static scan, helper-comment Option. gotcha).

## Codebase Patterns
- A `?` site is validated/lowered in THREE parallel layers that must agree:
  internal/analyze (lexical, what `goal check` runs via internal/check),
  internal/sema (the native check the interp gate runs), and internal/backend
  (the `?` destructure lowering). A `recv.M()?` callee resolves through the
  RECEIVER's type method set, not by name: analyze reads the receiver var's
  declared type lexically (analyze/methods.go scopeVarType) and the backend reads
  it from a per-function recvTypes map (built in funcDecl from d.Recv + params).
  The method's resolved signature MUST carry its Result/Option MODE (stored as
  `Method.Return FuncSig` in both analyze and sema), or a `Result[T,error]`
  method `?` mis-resolves as a non-error callee. Both concrete (`Methods`) and
  interface (`Interfaces`) method sets are consulted.
- A `sealed interface X {…}` is a distinct decl node: `*ast.SealedInterfaceDecl`
  in sema (NOT a `type X interface`), and a `sealed interface` token sequence in
  analyze (NOT caught by the `type`-keyed analyzeTypeDecls). Its method set must
  be resolved into Info.Interfaces / Tables.Interfaces explicitly — sema only
  flagged it in Info.Sealed and analyze only set Tables.Sealed before US-003, so
  anything keyed off the interface's METHODS (a `?` on a sealed-iface method, and
  potentially implements/hover) needs the body parsed, not just the name flagged.
- Value-position match lowering (enum AND Result/Option) reuses ONE set of
  position-parameterized emitters keyed by matchPos: enumMatch(m,pos,name),
  resultMatch/closedResultMatch/optionMatch(m,pos,name). Each wraps its arm bodies
  through armWrap (posStmt=bare, posReturn=`return <body>`, posVar=`name = <body>`)
  — renamed arms via armBodyRenamedWrap. The value-position DISPATCH lives in
  exactly three seams: returnStmt (posReturn), tryVarMatch (posVar, explicit `var
  name T`), tryAssignMatch (posVar, type via inferMatchType). A statement match
  goes through matchStmt with posStmt. To add value-position support for any new
  match subject kind, generalize its statement-position emitter to (m,pos,name) +
  armWrap and add a qualifier branch in those three dispatch sites — do NOT add a
  separate value-path. inferMatchType only resolves string/bool/enum arm bodies, so
  a `:=` (untyped) assignment whose arms aren't inferable keeps a located deferral;
  richer arm bodies need `var x T = match` or `return match`.
- AST-backend value-position lowering of Option construction rides on a SINGLE
  seam: the top of emitter.expr (emit.go). Every value position — a var/assign
  RHS, a call argument, a struct-literal field, and slice/map-literal elements —
  routes through expr(), so intercepting there (e.tryOptionValue) lowers
  Option.None->nil / Option.Some(ident)->&ident / Option.Some(other)->goalSome(other)
  everywhere at once. optionConstruction (lower.go) is the SINGLE classifier
  ("none"/"some-addr"/"some-box"); do not duplicate the Option-shape match. The
  pre-existing return/Result.Ok path (optionValueExpr/emitOptionReturn) handles the
  construction itself and never reaches expr(), so there is no double-lowering —
  leave it untouched.
- A backend helper func needed only in SOME outputs (e.g. the goalSome Option box)
  is emitted at the END of the file gated by a RUNTIME flag set when it is actually
  used (emitter.usedOptionHelper), NOT a static AST pre-scan. Go package-level
  decls are order-independent, so end-of-file placement is valid, and a runtime
  flag avoids the over-counting a static scan suffers (a scan cannot tell a
  value-position Some that boxes via goalSome from a return-position Some that the
  hoisted-temp path lowers, so it would inject an unused helper and break goldens).
  For package mode, surface the flag out of emitFileWith and emit ONE shared
  goal_options.go (mirrors the goal_prelude.go result-sum sharing).
- When asserting "no `Option.` token survived" in a backend test, beware the
  emitted helper's OWN doc comment: keep generated-helper comments free of the
  literal `Option.<Ident>` sequence or the substring check false-positives.
- The goscript interpreter validates input through the NATIVE sema aggregate
  only: Run()'s gate() runs sema.Check(file, info) before evaluating main and
  refuses on the FIRST sema.Error (Warnings are deferrals that do NOT block).
  sema.Check is the one native validation seam (exhaustive/fields/must-use/
  implements/question/closed/assert/convert) — any pre-run static guard reuses
  it. internal/interp must NEVER depend on go/types or internal/typecheck
  (TestInterpHasNoGoTypesOrTypecheckDep pins this via `go list -deps`).
- The goscript interpreter's host effects are CAPABILITY-MEDIATED (US-023):
  Interp carries `caps cap.CapabilitySet` + a configurable `stdout io.Writer`,
  constructed via `New(file, info, opts ...Option)` (defaults cap.GrantAll() +
  os.Stdout; WithCapabilities/WithStdout override). Every stdout effect flows
  through `emitStdout(pos token.Pos, write func(io.Writer) error)` — it writes
  through ip.stdout only when caps.Has(cap.Stdout). An EFFECTFUL stdlib shim
  (fmt.Println) cannot live in the pure package-level hostFuncs map — it is
  intercepted by key in evalHostCall and routed through the gate (passing
  sel.Pos()); pure value shims stay in the registry. US-024 made denial REAL: the
  not-granted branch returns a LOCATED, NAMED CapabilityError{Cap,Pos} (a returned
  typed error, errors.As-matchable — NOT a panicSignal) and performs no write.
  Future effects (time/env/file/net) reuse the same emit-gate pattern: gate on
  caps.Has(cap.X), else return CapabilityError{Cap: cap.X, Pos: effectPos}. cap is
  dependency-free, so it does not breach the US-022 dependency gate. NOTE token.Pos
  is a struct ({Offset,Line,Col}), not an int — use token.Pos{} when unavailable.
- PARSER: no semicolon is inserted after a BARE `return` before a newline, so a
  `=> return` match arm followed by another arm swallows the next arm's pattern
  ("expected IDENT, found =>"). In a multi-arm fixture use a block-body arm
  (`=> {}`) or `return <value>`; a bare-return arm only parses as the LAST arm.
- The goscript interpreter (internal/interp) evaluates NO top-level `const`/iota
  bindings yet, and sema.Info carries no defined-type/alias table — so an interp
  test program cannot reference an enum const (e.g. `RoleMember` is "undefined"
  at runtime), and a runtime zero of a defined type over a primitive
  (`type Role int`) resolves through interp.zeroValue to NilVal, not IntVal(0).
  When writing interp test fixtures, set such fields with plain literals or
  default them; if a story ever needs defined-type underlyings, resolve them
  from ip.file's TypeSpecs the way internal/backend buildTypeDecls does.

    as token.SEMICOLON and consumed where the grammar expects it; a stray one is
    an EmptyStmt.
  - COMPOSITE-LITERAL vs BLOCK ambiguity is the load-bearing gotcha: with no
    semicolons, `ident {` at statement level is read as a composite literal. The
    fix is go/parser's exprLev: parser.exprLev < 0 while parsing a control-clause
    header suppresses the brace-composite in parsePostfix, so a trailing body `{`
    is taken as the block. Bracketed contexts (parens/index/call/composite elts)
    do exprLev++ so composites are re-enabled inside them (`if check(T{})`). A
    statement whose final expression is a bare Ident/Selector/Index that is then
    immediately followed by a standalone block `{...}` is STILL ambiguous at
    exprLev 0 — order test inputs so a block follows a literal/call-terminated
    statement, not a bare-ident one. This is inherent to the tokenization until
    newline-aware lexing or `?`/full-expr work changes it.
  - for/if/switch headers share a pattern: set exprLev=-1, parse a simple stmt;
    if a ';' follows it was the Init (cond/tag parsed next), else the simple stmt
    WAS the cond/tag (extract via stmtExpr, which pulls X out of an ExprStmt);
    restore exprLev before parsing the body block.
  - Range detection lives in parseSimpleStmt(allowRange): after the assign token
    (:= or =), a `range` keyword yields *ast.RangeStmt (Key/Value from the lhs
    list) instead of an AssignStmt. parseForStmt passes allowRange=true.
  - internal/parser still imports only lexer+token+ast; tests stay package parser
    (internal). US-019 layers full Pratt expressions + postfix ? onto the same
    expression parser (parseExpr/parseOperand/parsePostfix) used here.

  - The parser strips // and /* comments, so any fix/fmt rule that must NOT drop a
    comment has to scan the raw source byte range (spanHasComment over token.Pos.Offset),
