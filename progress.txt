# Progress Log — goal AST front-end rewrite

## Codebase Patterns
- Zero-dependency: stdlib only. Tests use stdlib `testing` (NO testify), even
  though the injected go.md rule mentions testify — the project constraint wins.
- LSP go-to-definition (US-048) introduces the AST SYMBOL GRAPH the later editor
  stories (US-049 hover, US-050 references/rename) build on. internal/lsp/
  definition.go resolveDefinition parses (parser.ParseFile), builds a name-keyed
  declIndex (funcs/types/variants[enum][tag] -> declaration Range) over
  file.Decls BEFORE walking references (so a forward call resolves), then
  collectRefs (refVisitor, keyed by structural parent exactly like
  semantictokens roleVisitor) records each reference identifier's byte span ->
  target declaration Range. The handler maps the 0-based cursor to a byte offset
  with offsetForPosition (the inverse of check.OffsetToPosition — none existed in
  the tree) and returns the ref whose [start,end) covers it. Nil *Location
  marshals to JSON null, the best-effort contract shared with documentSymbol /
  semanticTokens (unknown URI / no symbol / unparseable -> null, never an error).
  Variants are keyed UNDER their enum because a variant tag is ALSO its own
  declaration ident; tests must locate the reference occurrence, not the first
  (declaration) occurrence. Resolution is single-document — locals, params,
  qualified imports, and cross-file are out of scope for this milestone.
- New packages live under `internal/`; fixtures go in a sibling `testdata/` dir
  (see internal/{analyze,check,pipeline,corpus}/testdata).
- Verify gates (prd.json verifyCommands): `go build ./...`, `go vet ./...`,
  `go test ./... -count=1`. All must be green before committing.
- AST-driven tools that must PRESERVE source formatting (internal/fix, and the
  coming goal fmt / LSP edits) compute byte ranges from node Pos()/End() offsets
  and emit minimal scan.Replacement edits via scan.Splice (pure string utils, not
  lexing) — never print the AST (it reflows). The parser DROPS // and /* comments,
  so any "don't drop a comment" guard must scan the raw source byte range
  (spanHasComment over token.Pos.Offset), not the AST.
- Follow REWRITE-ARCHITECTURE.md phasing; §10 details Phases 0–2 (the approved
  scope). Phase 0 = corpus hardening (US-001..US-010).
- Each story runs through the morse-code `feature` workflow in --automode on the
  current branch (no new branch); human-review gates are passed via
  `morse-code workflow resume`.
- The golden corpus is indexed by `corpus/manifest.json` (repo root). Regenerate
  after adding/removing golden files: `go generate ./internal/corpus` (or
  `go run ./cmd/corpus-gen -root .`). Paths are repo-root-relative, slash-form.
- `internal/corpus` imports `internal/pipeline` and `internal/check`. Any test
  that delegates to a corpus runner must therefore live in an EXTERNAL `_test`
  package (`package pipeline_test` / `check_test`) to avoid an import cycle; an
  internal-package test importing corpus will not compile. Internal and external
  test packages coexist fine in one directory. From `internal/<pkg>`, repo root is
  `../..` and the manifest is `../../corpus/manifest.json` (same depth as corpus).
- Corpus cases share Kind across modes: package-mode cases are also
  Kind=transpile. Any loop selecting transpile cases for a FILE-mode runner
  (one that reads Case.Input as a single file) MUST also guard
  `c.Mode == ModeFile`, or it will misread a package fixture's directory Input.
  Package cases run through corpus.RunPackage (Mode=package) only.
- internal/parser's newParser strips ONLY token.COMMENT; token.DOC_COMMENT (`///`)
  is RETAINED in the token stream and collected before each top-level decl
  (collectDoc) to attach as ast.FuncDecl.Doc. Any new parser loop that scans
  tokens must skip a stray DOC_COMMENT (parseBlock already does) — they otherwise
  only appear at top level before a declaration.
- parser.parseIndexSuffix is the SINGLE index entry point for BOTH type position
  (typeNameFrom) and expression position (parsePostfix); it parses a comma list →
  ast.IndexExpr (1 elt) or ast.IndexListExpr (>1). parseOperand also accepts
  type-literal starts (LBRACK/MAP/STRUCT) so parseExpr can parse type args like
  []byte and so `[]byte(p)` conversions / `map[K]V{}` composites parse; compositeOK
  must list every type form (ArrayType/MapType/StructType/IndexListExpr) that may
  precede a `{`. Enum payload-field colon is OPTIONAL (`name Type` or `name: Type`).
- ast.MatchArm.Body is a generic ast.Node: parseMatchArm dispatches `{`→block,
  statement-keyword→parseStmt (e.g. `=> return true`), else→parseExpr (value-pos
  arm `=> 1`). Don't force arm bodies to Expr.
- corpus may import internal/parser (nothing in corpus's deps — pipeline/check/
  project — imports parser), so the parse gate lives in package corpus
  (parse_runner.go: Parser/ParserFunc/RunParse + CaseInputs; TestParseGate). The
  corpus runner pattern is uniform: define an interface + Func adapter, drive the
  whole manifest, t.Fatalf on zero cases.
- ast.Sexpr (internal/ast/dump.go) renders any AST node to a deterministic,
  POSITION-FREE s-expression via reflection over exported fields (skips
  token.Pos, names token.Kind/FuncMod/ChanDir, elides zero/empty fields). It
  auto-covers new node types — adding an ast node needs NO dumper change. Keep
  node fields EXPORTED or they vanish from the dump. Parser structure is pinned
  by internal/parser TestASTSnapshots against testdata/snapshots/*.sexpr; after
  an INTENTIONAL grammar/AST change regenerate with
  `go test ./internal/parser -run TestASTSnapshots -update-snapshots` and review
  the diff before committing.
- The AST engine seam (Phase 2): internal/backend defines Backend{Emit(*ast.File,
  *sema.Info)(pipeline.Output,error)} + Formatter{Format([]byte)([]byte,error)} +
  GoFormatter (go/format) + the Transpile entry (parse->sema->Emit->format) that
  satisfies corpus.Transpiler. The Go emitter is the position-FREE, format-once
  model: emit.go writes only syntactically valid, token-correct Go (spaces between
  tokens, balanced braces) and lets GoFormatter normalize layout — so emit code
  need not pretty-print, only parse. Operators/keywords render via token.Kind's
  String() (e.g. d.Tok.String(), x.Op.String()), no token import needed in emit.go.
  Unsupported nodes (every goal-specific node, plus any not-yet-wired Go form)
  return a descriptive `backend: unsupported <kind> %T` error — later stories
  (US-032+) fill them in. internal/backend's test is EXTERNAL (package
  backend_test) so it can import internal/corpus (corpus does not import backend).
- backend/emit.go field separators are CONTEXT-sensitive: param/result/receiver
  lists are comma-joined (fieldList), but STRUCT FIELDS and INTERFACE METHODS must
  be newline/semicolon-separated (structType/interfaceType) — a comma there is a
  Go SYNTAX error that the format-once GoFormatter pass reports as "generated Go
  did not parse", not a layout nit. Interface methods render `Name(params)
  results` (funcSig without the `func` keyword); embedded interfaces emit the bare
  type. goal's parser has NO func-literal-as-expression (IIFE) and NO trailing
  variadic call spread (`f(xs...)`), so neither belongs in a plain-Go fixture.
- After changing cmd/goal subcommand/flag docs (guideCommands), regenerate the
  checked-in golden: `go run ./cmd/goal ai > AI-KNOWLEDGE-BOOTSTRAP.md`
  (TestBootstrapGoldenMatches fails otherwise). No CLI flag-drift test exists; the
  guide drift test only covers diagnostic codes.
- AST-backend goal-construct LOWERING is folded into internal/backend (no separate
  `lower` package — REWRITE-ARCHITECTURE marks lower/ir OPTIONAL, and emit.go does
  direct AST->Go-text). Per-construct encoders live in internal/backend/lower.go,
  mirrored VERBATIM from the legacy internal/pass/*.go (known-good build+vet-clean
  encodings) but read off sema facts instead of token scans. backend.Transpile
  MUST call sema.Resolve(file) (NOT sema.New(), which is empty) so the emitter has
  Enums/Sealed/Methods; the emitter stores `info *sema.Info` (nil-safe accessors
  enumOf/isSealed) + a local `pointerRecv` set (sema.Methods is star-stripped, so
  pointer-vs-value receiver is recomputed by walking File.Decls). US-034+ add more
  constructs the same way: encoder in lower.go, dispatch case in emit.go decl/expr.
- The three meanings of `Enum.Variant(x)` are distinct AST nodes, so emit-time
  lowering needs NO pass-ordering hack: a payload construction is *ast.VariantLit
  (parser builds it only for >=1 labeled arg), a DATA-LESS construction
  (`Status.Pending`) stays an *ast.SelectorExpr (lower it in the SelectorExpr case,
  GUARDED by info.Enums membership + VSet so io.Writer / receiver selectors are
  untouched), and a destructuring is *ast.VariantPattern (never emitted as a value).
  Recursive e.expr over VariantLit args lowers nested constructions for free.
- Function-scoped backend lowering rides on emitter.fnKind (set+restored around
  the body in funcDecl from resultOptionKind off the result type): a `return
  Ctor(...)` is only lowered when fnKind matches, the AST analogue of the splice
  passes' sigAt/funcSpans scoping. A statement-position match parses to ExprStmt{X:
  *MatchExpr} and is intercepted in stmt()'s ExprStmt case (NOT expr()); value-
  position match stays in expr() and still errors until US-036. Match-arm binding
  renames use emitter.renames (consulted in the Ident case, set per-arm via
  armBodyRenamed+defer delete); a generic ast.Node arm Body is dispatched by
  armBody() (ast.Stmt->stmt, ast.Expr->expr-statement). US-035/036/037 extend this
  same seam (encoder in lower.go + dispatch case in emit.go).
- A struct `implements` clause (StructType.Implements) lowers to a SEPARATE
  top-level decl emitted AFTER the type decl (implementsMarkers, called from decl()
  after genDecl) — never inside the struct. structType drops the clause and emits
  plain Go. Sealed iface -> marker METHOD `func (T) isI() {}`; ordinary iface ->
  `var _ I = T{}` (or `(*T)(nil)` when T has any pointer-receiver method). A
  qualified iface (io.Writer) is never sealed (sealed are in-file) so it routes to
  the var-assertion branch; render the iface name via typeExprName (Ident/Selector).
- The AST backend uses a SCOPE-AWARE gensym, not a magic `__goal_` prefix (US-035).
  emitFile seeds emitter.fileIdents = fileIdentSet(f) (every identifier in the file,
  lower.go). funcDecl opens a per-function scope: e.taken = e.newScope() (a fresh
  COPY of fileIdents) and, for an open-E Result function, mints the named returns
  `e.okName = e.gensym("ok")` / `e.errName = e.gensym("err")` BEFORE funcSig so the
  signature and body reference identical names; all of fnKind/okName/errName/taken
  are saved+restored (defer) so siblings start clean. e.gensym(want) returns want,
  or want+N for the least N avoiding any in-scope name. CONSEQUENCE: a function whose
  source already binds `err` (e.g. result_int's strconv.Atoi) gets a non-colliding
  `err1` named return — the scope-aware win. Function-scoped names (Result returns)
  live on the emitter; LOCAL temps (statement-position match `v`/`err`/`o`, Option
  box `some`, Option `?` pointer `o`) are minted at point of use — a match may sit in
  a NON-Result function (e.g. `handle`), so its destructure names must be locals, not
  the enclosing function's returns. funcSig falls back to plain "ok"/"err" when
  e.okName=="" (a bodyless Result-returning interface method / func-type expr).
- Postfix `?` (ast.UnwrapExpr) is lowered at STATEMENT position only, intercepted in
  stmt(): AssignStmt with a single UnwrapExpr RHS (`name := expr?` / `_ := expr?`) and
  ExprStmt with an UnwrapExpr X (bare `expr?`). A Result `?` destructures the callee's
  LOWERED arity (e.calleeSig reads sema.Info.FuncSignatures[name]: a plain
  `func()error` callee -> 1 value -> `if err := f(); err != nil`, a Result callee ->
  2 -> `if _, err := f(); err != nil`) and on failure returns the function's own
  (okName, errName). An Option `?` mints a fresh pointer temp per site and returns
  nil. A `?` in a roNone function hits the descriptive "`?` outside Result/Option"
  error. A closed-E Result function (E != error) is its OWN roKind (roResultClosed),
  set in funcDecl from sema.FuncSig[name].Mode==ModeResultClosed (NOT resultOptionKind,
  which returns roNone for closed-E so funcSig emits the `Result[T,E]` type verbatim).
  closedT/closedE ride on the emitter (saved/restored like okName/errName) and feed the
  Ok/Err sum constructors + closed `?` propagation.
- Closed-E Result (feature 06) lowers to the §8.1 GENERIC SUM, not a native tuple:
  the `resultPrelude` (Result[T,E] iface + Ok/Err carriers + marker methods, in
  lower.go) is emitted ONCE per file by file(), after import decls (isImportDecl
  guard) and before the first other decl, gated by needsResultPrelude(info) (any
  func is ModeResultClosed). `Result.Ok/Err(X)` -> `Ok/Err[T,E]{Value: X}`; a closed
  `match` and closed `?` are type-switches over `Ok[T,E]`/`Err[T,E]` with a panicking
  §8.2 default. Closed-E match is dispatched on the SCRUTINEE CALLEE's mode
  (calleeMode==ModeResultClosed), NOT the enclosing function — the matching function
  may be void (e.g. `handle`). Closed `?` returns the enclosing function's own
  Err[closedT,closedE], applying the `from func` conversion
  (FromRegistry[[2]string{callee.E, caller.E}].Name) when the error types differ.
- A `from func` (ast.FuncFrom) emits as an ORDINARY Go function (the modifier is
  stripped): its src->tgt registration already lives in sema.FromRegistry (sema.Resolve
  routes from/derive to the registry, NOT FuncSignatures), so the emitted func is just
  a callable conversion. funcDecl's modifier guard allows FuncPlain+FuncFrom; FuncDerive
  still fails (US-039).
- `...defaults` (feature 08, §8.5) and `assert` (feature 10) lower in internal/backend
  the same encoder-in-lower.go + dispatch-in-emit.go way. `...defaults` is parsed as
  *ast.SpreadElement{X: Ident "defaults"} inside a *ast.CompositeLit, so route ALL
  composite literals through emitter.compositeLit (not the inline exprList): it expands
  a `...defaults` element via defaultEntries (one `name: zeroLit(type)` per omitted
  declared field; an unsafe zero — nil pointer/map/chan/func, sum type, method-iface —
  is a located e.fail). A non-`defaults` spread (`...derive`) is US-039. zeroLit/
  zeroSafety/baseType are mirrored VERBATIM from analyze.ZeroLit/internal/pass.zeroSafety
  into lower.go, but read a `typeDecls map[string]string` (emitter.buildTypeDecls walks
  File TypeSpecs: struct->"struct", interface->"interface", alias->exprText(underlying))
  + sema enum/sealed sets — the AST-native replacement for analyze.Tables.TypeDecls.
- emitter.exprText(x) renders an expr to text WITHOUT touching the main output: it uses
  a FRESH sub-emitter (sharing info/renames/typeDecls), NOT a swap of e.b. Swapping a
  used strings.Builder by value panics ("illegal use of non-zero Builder copied by
  value") via copyCheck — never `saved := e.b`. exprText powers the assert panic-message
  text and an alias's underlying-type text.
- `assert` (*ast.AssertStmt) lowers in stmt()'s new AssertStmt case (assertStmt):
  `if !(cond) { panic("assertion failed: "+condtext[ +": "+fmt.Sprintf(msg,args...)]) }`,
  condtext = exprText(Cond), the real cond re-emitted via e.expr. The printf-message form
  needs fmt: file() injects `import "fmt"` ONCE right after the package clause when
  needsFmtImport(f) (Walk finds any AssertStmt with Msg!=nil) && !importsPkg(f,"fmt").
  Imports must precede other decls, so inject before the closed-E prelude / first decl.
- `derive func` (feature 12, §12) lowers in internal/backend the same encoder-in-lower.go
  + dispatch-in-emit.go way; `from func` is ALREADY a plain func (funcDecl's FuncFrom case,
  US-037). funcDecl's d.Mod switch routes ast.FuncDerive -> deriveDecl, which OWNS the whole
  declaration (a derive has no ordinary body to emit) and returns BEFORE the Result/Option
  gensym-scope setup. deriveDecl reads the source name+type (first param) and the target
  type+fallibility from deriveTarget(d.Type.Results) (one result => total; (T,error) => the
  derive returns (T,error) and threads the error). A BODIED derive's overrides come from
  deriveOverrides: walk Body->ReturnStmt->CompositeLit.Elts; a *ast.KeyValueExpr{Key:Ident}
  is an override (value *ast.Ident "_" => SKIP, field left at its `var out` zero); the
  `...derive(src)` *ast.SpreadElement is the implicit fill marker (ignored — the unoverridden
  fields ARE the fill). genConversion emits `func name(src S) <ret> { var out T; <overrides>;
  <resolved fields>; return out[, nil] }`. resolveField is mirrored VERBATIM from
  internal/pass.resolveField but reads sema.Info.Structs (case-insensitive findSemaField) +
  sema.Info.FromRegistry: identity (sf==tf) `out.F=src.F`; registry total `out.F=conv(src.F)`;
  registry fallible `v,err:=conv(src.F); if err!=nil {return out,err}; out.F=v`; ptr/Option
  `*A->*B` nil-guarded re-address; slice `[]A->[]B` make+indexed loop; array `[N]A->[N]B`
  in-place; map `map[K]A->map[K]B`; nested in-file struct via deriveBody. An unsourced or
  unconvertible field is a LOCATED e.fail, never a silent zero (the footgun §12 kills).
  Override VALUES emit via e.exprText (AST expr, not string parse — FR-6). Temp names are
  scope-aware gensyms (one shared `err` per fallible derive; `v`/`p`/`i`/`k`/`s` per site),
  NOT `__goal_v0`. typeExprString (lower.go) renders a source/target type expr to the
  sema-canonical key string (matching Structs/FromRegistry keys). The string-level splitters
  (derefType/ptrInner/arrElem/mapKV/elemConv) port verbatim — they operate on already-resolved
  sema type strings. NOTE the foreign-package derive fixture (US-009) still runs through the
  SPLICE engine via corpus.RunPackage; this AST lowering covers only the file-mode features/12
  cases.

- internal/typecheck (the go/types depth checker, on the DEFAULT `goal build/check` path) transpiles
  internally — it now uses backend.TranspilePackage. The AST backend therefore must faithfully emit

  generic type-parameter DECLARATIONS (`type Box[T any]`) and type ALIASES (`type T = U`). The splice
  engine masked these by passing them through verbatim; the AST parser/emitter must handle them.
- Generic type-parameter DECLARATIONS are parsed in parser.parseTypeSpec via atTypeParams()
  (disambiguates `type T[P any]` from `[]T`/`[N]T`/`[3]T` arrays using kindAt(2): typeparams iff
  `[` then IDENT then COMMA-or-type-start) + parseTypeParams() (shared-constraint name lists like
  `[T, U any]`). They live on ast.TypeSpec.TypeParams *FieldList and emit via fieldList(fl,"[","]").
  Generic INSTANTIATION (`Box[int]`) already parsed via parseIndexSuffix (IndexExpr/IndexListExpr).
  kindAt(n) is the new n-token lookahead helper (peekKind == kindAt(1)).
- Type ALIASES carry on ast.TypeSpec.Assign token.Pos (zero == defined type). parser records the
  `=` position; emit.go's TypeSpec case emits `= ` when Assign is non-zero. Sexpr SKIPS token.Pos so
  AST snapshots are unaffected; but the BACKEND OUTPUT changed (`type T int64` -> `type T = int64`),
  so any corpus golden with an alias must be regenerated (-update-goldens). Aliases vs defined types
  have DIFFERENT go/types identity — emitting the alias faithfully is what makes the depth checker's
  signature-equivalence checks pass (TestImplementsAliasEqualSignatureIsNotAMismatch).
- scan.Splice/Replacement and the analyze fact tables are STILL LIVE (consumed by check/fix/lsp/
  typecheck/project). Nothing in scan/analyze was used SOLELY by the deleted splice passes, so the
  "dead scanning" can only be removed once US-044 (fix->AST) and US-046 (lsp->AST) migrate those
  consumers off the token scanner. Don't try to delete scan/analyze symbols until then.
- `goal fmt` (internal/goalfmt.Source) is a position-driven REINDENTER, NOT an AST
  pretty-printer: it parses with parser.ParseFile only as a validity gate, then rewrites
  RAW SOURCE LINES (normalizing leading indent = one tab per delimiter-nesting depth from
  lexer.Tokens, trailing whitespace, and blank-line runs) — never printing the bare AST
  (which would drop // and /* comments). Because only whitespace the lexer skips changes,
  the token stream/AST/meaning is invariant and it is idempotent by construction. Any
  future fmt enhancement (struct-field alignment, switch-case dedent) must keep that
  raw-source, position-driven discipline — do NOT switch to AST printing.
- The LSP document model is AST-derived (US-046). internal/lsp/symbols.go
  collectSymbols PARSES (parser.ParseFile) and walks ast.File.Decls — there is no
  more token scan (scanDecls is gone). Outline ranges come straight off node
  Pos()/End() offsets via rangeOf, so a bodyless decl never overruns the next
  (End() stops at its own last token). internal/lsp no longer imports internal/scan:
  diagnostics.go tokenEnds tokenizes via lexer.Tokens (token end = Pos.Offset +
  len(Lit)). BUT diagnostic FINDINGS still come from check.Analyze /
  AnalyzePackageInDirWith — the checker itself is still analyze/scan-based (its
  package path needs analyze.EnrichForeign), so don't expect sema there yet.
  rangeOf still uses check.OffsetToPosition, so symbols.go keeps the internal/check
  import even though the scan import is gone.
- LSP semantic tokens (US-047) are computed by overlaying an AST role map on the
  lexer token STREAM, not by walking the AST alone: lexer.Tokens gives ordered
  positions + lengths + the lexer-confident classes (keywords incl. match/enum,
  comments, string/char, int/float/imag, and the goal operators QUESTION/FAT_ARROW/
  ELLIPSIS), and astRoles (parser.ParseFile + ast.Walk roleVisitor) supplies the
  identifier ROLE keyed by Ident byte offset. roleVisitor fixes a name's role by its
  STRUCTURAL parent (handled at the EnumDecl/TypeSpec/FuncDecl/VariantLit/
  VariantPattern/CallExpr node, NOT at the bare Ident) and always returns itself so
  Walk still descends into bodies. An IDENT with no recorded role is NOT emitted —
  builtins like int stay grammar-coloured rather than mislabelled. The wire form is
  the LSP delta-encoded 5-tuple [dLine,dStartChar,length,type,modifiers], 0-based via
  check.OffsetToPosition; tokenLen falls back to len(Kind.String()) for empty-Lit
  operators. The legend ordering is one source of truth: the sem* index constants
  index semanticTokenTypes, shared by the advertised SemanticTokensProvider and the
  encoder, so wire indices can't drift. GOTCHA: a `?` inside a string ("?") lexes as
  STRING, not QUESTION — a test locating the postfix `?` by substring must avoid a
  stray `?` string in its sample.

## [2026-06-27T17:30:00Z] - US-048
- Added textDocument/definition to the goal LSP, resolved from the AST symbol
  graph of the open document. A function/method call, an enum-variant reference
  (construction value, match pattern, or Enum.Variant selector), and a
  type/enum/sealed name reference each resolve to the range of their declaration;
  an unknown URI, a position over no resolvable symbol, or unparseable source
  yields a null result (best-effort, like documentSymbol/semanticTokens).
- Implementation: build a name-keyed declaration index over file.Decls, then walk
  the AST recording each reference identifier's byte span -> declaration Range
  (refVisitor keyed by structural parent, mirroring the semantic-token role
  visitor). The index is built before references so a forward call resolves. The
  cursor (0-based line/char) maps to a byte offset via offsetForPosition (inverse
  of check.OffsetToPosition); the ref whose [start,end) covers the offset wins.

- Learnings for future iterations:
