## Codebase Patterns
- Generated Go is committed under internal/**/*.go and gated by `task check` →
  `verify-generated`: it re-emits to a scratch tree and diffs against the tree.
  Any change to emit output requires re-running `task generate` so the committed
  files match, or the drift gate fails.
- `cmd/goal/main.go` is hand-written Go (NOT goal-generated). Edit it directly,
  then `task build` rebuilds bin/goal before `task generate` regenerates internal.
- The emit write path is `emitFiles` in cmd/goal/main.go; corpus/by-example tests
  compare in-memory pipeline output (gf.Go), not on-disk emitted files, so a
  write-time change (e.g. a header) does not affect those goldens.
- The project verify command is `task check` (vet + build + verify-generated +
  go test ./...). Zero-dependency: stdlib `testing` only, no testify.
- morse-code `history/` is gitignored — initiative artifacts are not committed.
